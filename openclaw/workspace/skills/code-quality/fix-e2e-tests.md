# Fix E2E Test Failures

## Workflow

1. **Run the failing E2E test** to reproduce
2. **Read the test** to understand the user flow being tested
3. **Check if it's a flaky test**: Run it 2-3 times. If it passes intermittently, it's flaky.
4. **For flaky tests**: Fix the timing/race condition, don't add retries
5. **For deterministic failures**: Trace to source code or test setup issue
6. **Fix and verify**
7. **Create PR**

## Common E2E Failure Categories

### Selector Failures
The most common E2E failure — an element selector no longer matches.

**Diagnosis:**
- Component was renamed or restructured
- CSS class changed
- `data-testid` attribute was removed or renamed

**Fix:**
- Update the selector to match current DOM
- Prefer `data-testid` over CSS classes or text content
- If the component was intentionally restructured, update the test flow

### Timing / Race Conditions
Tests fail because they don't wait for async operations.

**Fix patterns:**
```typescript
// Playwright
await page.waitForSelector('[data-testid="result"]')
await expect(page.locator('[data-testid="result"]')).toBeVisible()

// Cypress
cy.get('[data-testid="result"]', { timeout: 10000 }).should('be.visible')
```

**Never add arbitrary waits:**
```typescript
// BAD - flaky and slow
await page.waitForTimeout(5000)

// GOOD - wait for specific condition
await page.waitForResponse(resp => resp.url().includes('/api/data'))
```

### API/Network Failures
Backend returned unexpected response during E2E run.

**Diagnosis:**
- Is the API endpoint correct?
- Did the API contract change?
- Is test data setup correct?

**Fix:**
- Update API calls to match current contract
- Fix test data fixtures
- If API is genuinely broken, fix the API code (not the test)

### Authentication Failures
Login flow fails in E2E test.

**Diagnosis:**
- Test credentials expired or changed
- Auth flow UI changed
- Session/cookie handling broken

**Fix:**
- Update test credential configuration
- Update login page selectors/flow
- Check cookie/session setup in test fixtures

### State Pollution
Previous test leaves state that breaks subsequent tests.

**Diagnosis:**
- Test passes in isolation but fails in suite
- Order-dependent failures

**Fix:**
- Add proper cleanup in `afterEach` / `afterAll`
- Reset database state between tests
- Clear localStorage/sessionStorage
- Never rely on other tests' side effects

## Verification

Run the specific failing test first, then the full suite:
```bash
# Playwright
npx playwright test path/to/test.spec.ts
npx playwright test

# Cypress
npx cypress run --spec path/to/test.cy.ts
npx cypress run
```

## PR Format

```
## E2E Test Fix

### Failure
- Test: `test name`
- Error: `brief error message`

### Root Cause
[Why the E2E test was failing — selector change, timing, API, etc.]

### Fix
[What was changed and why]

### Verification
- Specific test passes: yes
- Full E2E suite passes: yes

---
*Generated by Bumblebee via OpenClaw*
```

## Never Do

- Add retry logic to mask flaky tests
- Add `test.skip()` to make the suite pass
- Add arbitrary `sleep()` / `waitForTimeout()` calls
- Weaken assertions to accept any result
- Delete E2E tests
