# Fix Linter Violations

## Workflow

1. **Run linter in check mode** to get full violation list
2. **Apply auto-fix** where available
3. **Manually fix** violations auto-fix can't handle
4. **Verify** clean lint pass
5. **Create PR** with all fixes

## TypeScript (ESLint)

### Auto-fixable
Run first — these are safe:
```bash
npx eslint --fix .
```

### Manual Fix Patterns

**Unused variables/imports:**
- Remove if truly unused
- If re-exported from `index.ts`, add to explicit export list

**Missing return types:**
```typescript
// Bad
export function getData() { return fetch('/api') }

// Good
export async function getData(): Promise<Response> { return fetch('/api') }
```

**Unsafe `any` usage:**
```typescript
// Bad
function process(data: any) { return data.name }

// Good
function process(data: unknown): string {
  if (typeof data === 'object' && data !== null && 'name' in data) {
    return String((data as { name: unknown }).name)
  }
  throw new Error('Invalid data')
}
```

**React hooks rules:**
- Move hooks above all conditionals and early returns
- Extract conditional logic into the hook's callback, not around the hook call

**No console.log in production:**
- Replace with proper logger or remove
- If debug logging, use `console.debug` (if project allows) or remove

### Never Do
- Add `// eslint-disable-next-line` or `/* eslint-disable */`
- Change ESLint configuration to relax rules
- Add `@ts-ignore` to bypass type issues revealed by lint

## Python (Ruff)

### Auto-fixable
```bash
ruff check --fix .
ruff format .
```

### Manual Fix Patterns

**Complexity (C901):**
- Extract helper functions from complex methods
- Replace nested conditionals with early returns
- Use guard clauses

**Naming conventions (N801/N802):**
- Follow existing project style
- Check `per-file-ignores` for intentional exceptions

**Type annotation missing (ANN):**
```python
# Bad
def process(data):
    return data["name"]

# Good
def process(data: dict[str, Any]) -> str:
    return data["name"]
```

### Never Do
- Add `# noqa` comments to suppress violations
- Modify `pyproject.toml` ruff configuration
- Disable rules project-wide

## PR Format

```
## Linter Fixes

### Auto-fixed
- [list of auto-fixed rule violations]

### Manually Fixed
- [list of manually fixed violations with brief explanation]

### Verification
- `eslint .` or `ruff check .` exits clean
- No suppressions added

*Generated by Bumblebee via OpenClaw*
```
