# Fix CI Failures

## Workflow

When a CI failure is detected on an enrolled repo:

1. **Fetch logs**: `gh run view RUN_ID --repo OWNER/REPO --log-failed`
2. **Classify failure**: test | lint | type | build | workflow
3. **Check retry count**: If >= 2 previous auto-fix PRs, escalate instead
4. **Clone repo**: `gh repo clone OWNER/REPO /tmp/fix-workspace/REPO`
5. **Create fix branch**: `git checkout -b "auto-fix/$(date +%s)"`
6. **Diagnose root cause**: Trace error to source file and line
7. **Implement minimal fix**: Smallest change that resolves the failure
8. **Verify**: Run stack-appropriate test/lint/type commands
9. **Submit PR**: If verification passes
10. **Notify**: Telegram message with outcome and PR link

## TypeScript Stack

### Common Patterns

**Vitest Test Failures:**
- Import path errors: Check case sensitivity, missing extensions, incorrect relative paths
- Mock setup: Ensure `vi.mock()` configured before imports, `vi.fn()` return types match
- Async handling: Missing `await`, unresolved promises, `act()` wrapper in React tests
- Snapshot mismatches: Run `npx vitest run --update` only if component change was intentional

**ESLint Errors:**
- Unused imports: Remove import or use the value
- Missing return types: Add explicit annotations to exported functions
- Unsafe `any`: Replace with proper types, use `unknown` + type guards for dynamic types
- React hooks rules: Hooks at top level only, not inside conditions/loops

**TypeScript Errors:**
- Strict null checks: Add null/undefined checks, use `?.` and `??`
- Generic type mismatches: Verify type parameters between signatures and call sites
- Missing return types: Add explicit returns where inference fails
- Module resolution: Verify path aliases match import paths

**Build Errors:**
- Circular dependencies: Trace import chain, extract shared types to separate file
- Missing exports: Ensure symbols properly exported from source modules
- Dynamic import issues: Verify `next/dynamic` has loading/error boundaries
- Environment variables: `process.env.NEXT_PUBLIC_*` for client-side only

### Verification Commands
```bash
npx vitest run --reporter=verbose    # Tests
npx eslint .                          # Lint
npx tsc --noEmit                      # Types
npm run build                         # Build
```

## Python Stack

### Common Patterns

**Import Errors:**
- ModuleNotFoundError: Check package name vs import name (e.g., `pip install Pillow` → `import PIL`)
- Circular imports: Break with lazy imports inside functions or extract shared types
- Relative vs absolute: Use absolute imports unless project consistently uses relative
- Missing `__init__.py`: Verify in all package directories

**Fixture Issues:**
- conftest.py not discovered: Check directory placement relative to tests
- Scope mismatch: `function` for isolation, `session` for speed
- Missing dependencies: Ensure fixture B is in same or parent conftest if A depends on it
- Autouse side effects: Check if `autouse=True` interferes with specific tests

**Async/Await:**
- Missing `await`: Look for `<coroutine object>` in error output
- pytest-asyncio mode: Check `auto` vs `strict` — affects marker requirements
- Mixing sync/async: Use `asyncio.to_thread()` for blocking in async context
- Event loop already running: Use `pytest-asyncio` fixtures, not `asyncio.run()`

**Pydantic v2 Migration:**
- Deprecated validators: `@root_validator` → `@model_validator(mode='before')`
- Deprecated methods: `.dict()` → `.model_dump()`, `.json()` → `.model_dump_json()`
- Field config: Inner `class Config` → `model_config = ConfigDict(...)`

**Ruff Lint:**
- F401 unused imports: Remove or add to `__all__` for re-exports
- I001 import sorting: `ruff check --fix --select I .` (always safe)
- E501 line too long: Break with parenthesized expressions or intermediate variables

### Verification Commands
```bash
pytest -x -v                  # Tests
ruff check .                  # Lint
mypy .                        # Types
ruff format --check .         # Format
```

## Escalation Template

After 2 failed attempts, create a `needs-human` issue:
```
## Auto-Fix Escalation
**Status:** 2 fix attempts failed — needs human review

### Failure Context
- Branch: `BRANCH`
- CI Run: link

### Previous Fix Attempts
- PR #N: link

### CI Failure Logs
[Sanitized logs in collapsible section]
```
