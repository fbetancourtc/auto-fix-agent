# Fix Code Coverage Gaps

## Workflow

1. **Run tests with coverage** to identify gaps
2. **Identify uncovered files/functions** — prioritize by importance
3. **Write tests** for uncovered code paths
4. **Run coverage again** to verify improvement
5. **Create PR** with new tests

## Generating Coverage Report

### TypeScript
```bash
npx vitest run --coverage
# Look for: Uncovered lines, branches, functions in output
```

### Python
```bash
pytest --cov --cov-report=term-missing
# Look for: "Missing" column showing uncovered line numbers
```

## Prioritization

Focus on high-value coverage first:

1. **Public API functions** — most impact, most likely to regress
2. **Business logic** — core domain behavior
3. **Error handling paths** — often uncovered but critical
4. **Edge cases** — null inputs, empty arrays, boundary values
5. **Utility functions** — lower priority unless widely used

## Writing Tests

### Principles

- **Test behavior, not implementation** — assert on outputs and side effects
- **One assertion per test** (preferred) — clear failure messages
- **Descriptive names** — `it('should return empty array when no items match filter')`
- **Arrange-Act-Assert** — clear structure in every test
- **No mocking unless necessary** — prefer real implementations for unit tests

### TypeScript Test Template
```typescript
describe('ModuleName', () => {
  describe('functionName', () => {
    it('should handle the happy path', () => {
      // Arrange
      const input = createTestInput()

      // Act
      const result = functionName(input)

      // Assert
      expect(result).toEqual(expectedOutput)
    })

    it('should handle null input', () => {
      expect(() => functionName(null)).toThrow('Input required')
    })

    it('should handle empty array', () => {
      const result = functionName([])
      expect(result).toEqual([])
    })
  })
})
```

### Python Test Template
```python
class TestModuleName:
    def test_happy_path(self):
        # Arrange
        input_data = create_test_input()

        # Act
        result = function_name(input_data)

        # Assert
        assert result == expected_output

    def test_none_input(self):
        with pytest.raises(ValueError, match="Input required"):
            function_name(None)

    def test_empty_list(self):
        result = function_name([])
        assert result == []
```

## Test Placement

- **TypeScript**: Place in `tests/` or `__tests__/` mirroring source structure
- **Python**: Place in `tests/` mirroring source structure
- Follow existing test file naming:
  - TypeScript: `module.test.ts` or `module.spec.ts`
  - Python: `test_module.py`

## PR Format

```
## Coverage Improvement

### Before
- Overall: X%
- Uncovered files: [list]

### Tests Added
- `tests/file.test.ts`: Tests for [module] covering [scenarios]
  - happy path
  - error handling
  - edge cases

### After
- Overall: Y% (+Z%)
- Newly covered: [list of functions/files]

### Verification
- All new tests pass
- No existing tests broken
- Coverage improved by Z%

---
*Generated by CodeQual via OpenClaw*
```

## Never Do

- Write tests that assert on implementation details (private methods, internal state)
- Write tests that always pass regardless of behavior (empty assertions)
- Add `istanbul ignore` or `coverage: ignore` comments
- Delete code to improve coverage percentage
- Write snapshot tests as a lazy coverage boost
