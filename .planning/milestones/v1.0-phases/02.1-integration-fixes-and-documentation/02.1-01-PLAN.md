**1a. Fix validate-diff.sh diff target (CRITICAL — FIXG-02, FIXG-03)**

In `scripts/validate-diff.sh`, line 36, change the diff detection from unstaged/staged (empty after commit) to committed diff:

Replace:
```bash
# Get list of changed files (staged and unstaged)
CHANGED_FILES=$(git diff --name-only HEAD 2>/dev/null || true)
STAGED_FILES=$(git diff --cached --name-only 2>/dev/null || true)
```

```bash
    git checkout HEAD -- "$file" 2>/dev/null || true
```

**1b. Wrap circuit breaker getWorkflowRun in try/catch (MEDIUM — SECR-02)**

In `.github/workflows/auto-fix.yml`, the circuit breaker step (step 4, ~line 80), the `getWorkflowRun` API call is unguarded. A network error would crash the entire workflow instead of failing open.

Wrap lines 80-85 (the `getWorkflowRun` call and `headBranch` extraction) in a try/catch that fails open:

Replace the circuit breaker script section:
```javascript
            // Get the workflow run details
            const run = await github.rest.actions.getWorkflowRun({
              owner,
              repo: repoName,
              run_id: runId
            });

            const headBranch = run.data.head_branch;
```

With:
```javascript
            // Get the workflow run details
            // Fail open: if API call fails, proceed with fix (don't block on transient errors)
            let headBranch;
            try {
              const run = await github.rest.actions.getWorkflowRun({
                owner,
                repo: repoName,
                run_id: runId
              });
              headBranch = run.data.head_branch;
            } catch (e) {
              core.warning(`Circuit breaker: getWorkflowRun failed (failing open): ${e.message}`);
              core.setOutput('should_proceed', 'true');
              return;
            }
```
