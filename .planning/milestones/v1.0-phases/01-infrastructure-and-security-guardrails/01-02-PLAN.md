Key security patterns from research:

1. **Input sanitization (SECR-01)**: Environment variable indirection (never inline `${{ }}`); regex strip of injection markers; truncation to 500 lines; secret pattern redaction (ghp_, ghs_, github_pat_).

2. **Circuit-breaker (SECR-02)**: Defense-in-depth with 3 layers:
   - GITHUB_TOKEN pushes don't trigger workflow_run (built-in, but we use App token so this doesn't help us)
   - Branch name check: skip if branch starts with `auto-fix/`
   - PR label check: skip if any open PR on the branch has the `auto-fix` label

3. **Token limits (SECR-03)**: Two controls:
   - `--max-turns 10` in claude_args (soft cap: agent stops after 10 tool exchanges)
   - `timeout-minutes: 15` on the job (hard cap: kills job after 15 min)
   - Combined worst case: ~$5.00 per run

4. **Scope restriction (SECR-04)**: Two controls:
   - `--allowedTools` in claude_args: Edit, Read, Write, Bash(limited commands only)
   - No secrets passed to agent step environment (anthropic_api_key is passed via action input, not env)
   - Workflow permissions scoped to only `contents: write`, `pull-requests: write`, `actions: read`
   - NOTE: `--allowedTools` may have enforcement gap (issue #860). Post-run diff validation is added as defense-in-depth (Phase 2 implements full diff check; this plan adds the `--allowedTools` restriction).

Anti-patterns to avoid:
- NEVER use inline `${{ }}` with untrusted content in `run:` blocks
- NEVER pass GITHUB_TOKEN or App token to agent step's env
- NEVER use `secrets: inherit` cross-org
</context>

**1. Create `scripts/sanitize-logs.sh`:**

```bash
#!/usr/bin/env bash
set -euo pipefail

# Usage: sanitize-logs.sh <input_file> <output_file>
# Strips prompt injection patterns and secrets from CI logs before agent ingestion.

INPUT_FILE="${1:?Usage: sanitize-logs.sh <input> <output>}"
OUTPUT_FILE="${2:?Usage: sanitize-logs.sh <input> <output>}"
```

Implement the following sanitization steps in order:

a. **Truncate** to last 500 lines: `tail -500 "$INPUT_FILE" > "$OUTPUT_FILE"`

b. **Strip prompt injection patterns** (use `sed -i` on OUTPUT_FILE):
   - `-- Additional ... instruction --` pattern (case-insensitive)
   - `IMPORTANT: You must` / `IMPORTANT: Please` / `IMPORTANT: Always` patterns
   - Role assumption: `You are a ...`, `Act as ...`, `Ignore previous ...`, `Ignore all previous ...`, `Disregard ... instructions ...`
   - System prompt override: `<system>`, `</system>`, `[SYSTEM]`, `[/SYSTEM]`

c. **Strip shell injection patterns**:
   - Backtick command substitution: `` `...` ``
   - Dollar-paren substitution: `$(...)`

d. **Strip YAML frontmatter blocks** that could override system prompt: remove `---` to `---` blocks

e. **Redact GitHub token patterns**:
   - `ghp_` tokens (classic PAT)
   - `ghs_` tokens (installation tokens)
   - `github_pat_` tokens (fine-grained PAT)
   - Generic `sk-` patterns (API keys)

f. **Redact common secret patterns**:
   - Lines containing `ANTHROPIC_API_KEY=`, `AWS_SECRET_ACCESS_KEY=`, `GITHUB_TOKEN=` -- redact the value portion

g. Add a final line count echo: `echo "Sanitized: $(wc -l < "$OUTPUT_FILE") lines"`

Make the script executable: `chmod +x scripts/sanitize-logs.sh`

**2. Update `.github/workflows/auto-fix.yml` -- Circuit-breaker step:**

Replace the placeholder circuit-breaker step (from Plan 01) with the full implementation using `actions/github-script@v7`:

```yaml
- name: Circuit breaker
  id: circuit
  uses: actions/github-script@v7
  with:
    github-token: ${{ steps.app-token.outputs.token }}
    script: |
      const repo = '${{ inputs.repository }}'.split('/');
      const owner = repo[0];
      const repoName = repo[1];
      const runId = parseInt('${{ inputs.failed_run_id }}');

      // Get the workflow run details
      const run = await github.rest.actions.getWorkflowRun({
        owner,
        repo: repoName,
        run_id: runId
      });

      const headBranch = run.data.head_branch;

      // Check 1: Is this from an auto-fix branch?
      if (headBranch && headBranch.startsWith('auto-fix/')) {
        core.setOutput('should_proceed', 'false');
        core.info(`Circuit breaker TRIPPED: branch "${headBranch}" is an auto-fix branch`);
        return;
      }

      // Check 2: Does any open PR on this branch have the auto-fix label?
      try {
        const pulls = await github.rest.pulls.list({
          owner,
          repo: repoName,
          head: `${owner}:${headBranch}`,
          state: 'open'
        });
        for (const pr of pulls.data) {
          if (pr.labels.some(l => l.name === 'auto-fix')) {
            core.setOutput('should_proceed', 'false');
            core.info(`Circuit breaker TRIPPED: PR #${pr.number} has auto-fix label`);
            return;
          }
        }
      } catch (e) {
        core.warning(`Circuit breaker: could not check PRs: ${e.message}`);
        // Fail open -- proceed if we can't check (better to fix than to skip)
      }

      core.setOutput('should_proceed', 'true');
      core.info('Circuit breaker: CLEAR -- proceeding with fix');
```
