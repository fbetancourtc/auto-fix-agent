```yaml
name: Auto Fix Reusable
on:
  workflow_call:
    inputs:
      app_id:
        required: true
        type: string
        description: "GitHub App ID for cross-org token generation"
      failed_run_id:
        required: true
        type: string
        description: "ID of the failed workflow run to analyze"
      repository:
        required: true
        type: string
        description: "Full repo name (owner/repo) where CI failed"
    secrets:
      anthropic_api_key:
        required: true
      app_private_key:
        required: true
```

The `jobs:` section should contain a single job `fix:` with:
- `runs-on: ubuntu-latest`
- `timeout-minutes: 15`
- `permissions:` with `contents: write`, `pull-requests: write`, `actions: read`

Steps (implement as placeholder structure -- security steps will be filled by Plan 02):
1. **Generate token** -- `actions/create-github-app-token@v2` with `app-id: ${{ inputs.app_id }}`, `private-key: ${{ secrets.app_private_key }}`. Do NOT set `owner:` explicitly -- it defaults to `github.repository_owner` which dynamically targets the calling org.
2. **Checkout** -- `actions/checkout@v4` with `repository: ${{ inputs.repository }}`, `token: ${{ steps.app-token.outputs.token }}`
3. **Circuit breaker** -- Add comment `# Circuit breaker -- implemented in Plan 01-02` and a placeholder step using `actions/github-script@v7` with `script: core.setOutput('should_proceed', 'true');` (Plan 02 fills real logic)
4. **Get failed logs** -- Guarded by `if: steps.circuit.outputs.should_proceed == 'true'`. Uses `env:` indirection (NOT inline `${{ }}`): set `GH_TOKEN`, `RUN_ID`, `REPO` as env vars. Run: `gh run view "$RUN_ID" --repo "$REPO" --log-failed 2>/dev/null | tail -500 > /tmp/ci-logs.txt || echo "No logs available" > /tmp/ci-logs.txt`. Add comment `# Sanitization -- implemented in Plan 01-02`.
5. **Load prompt** -- Read `config/repo-stack-map.json` to determine stack, then `cat prompts/${STACK}.md`. Use `GITHUB_OUTPUT` heredoc pattern to pass multi-line prompt output. Handle missing stack gracefully (default to `typescript`).
6. **Run Claude Code Action** -- `anthropics/claude-code-action@v1` with `anthropic_api_key`, `prompt` (combining loaded prompt + CI logs from file), `claude_args` with `--max-turns 10` and `--allowedTools` placeholder. Add comment `# Token limits and scope restriction -- configured in Plan 01-02`.

CRITICAL: Never use inline `${{ }}` with untrusted content (CI logs, PR titles). Always use `env:` indirection for shell commands.
