<task type="auto">
  <name>Task 2: Create main webhook handler with environment gate, signature verification, and waitUntil shell</name>
  <files>api/webhook.ts</files>
  <action>
Create api/webhook.ts as the Vercel serverless function entry point using the Web API handler format (`export default { async fetch(request: Request) }`).

The handler MUST follow this exact flow (per user decisions in CONTEXT.md):

1. **Import Sentry first** -- `import './lib/sentry.ts'` as the very first import (ensures Sentry.init runs at module scope before anything else). Then import Sentry separately for captureException usage.
2. **Import** waitUntil from @vercel/functions, verifyWebhookSignature from ./lib/verify, extractHeaders from ./lib/types, flushSentry from ./lib/sentry, and * as Sentry from @sentry/node.

3. **Environment gate (first check):** If `process.env.VERCEL_ENV !== 'production'`, return `new Response('', { status: 404 })`. Per user decision: return 404, NOT 403 -- reveals nothing about the endpoint.

4. **Method gate:** If `request.method !== 'POST'`, return `new Response('', { status: 405 })`.

5. **Read raw body:** `const rawBody = await request.text()` -- MUST be done before any other body read (per Pitfall 2 in RESEARCH.md).

6. **Extract headers:** Use extractHeaders() to get signature, eventType, deliveryId.

7. **Verify HMAC-SHA256 signature:** Call verifyWebhookSignature(rawBody, signature, secret). If invalid, return `new Response('', { status: 401 })`. Secret comes from `process.env.GITHUB_WEBHOOK_SECRET ?? ''`.

8. **Respond 200 immediately:** `const response = new Response('OK', { status: 200 })`.

9. **Defer processing:** `waitUntil(processEvent(rawBody, headers))` -- where headers is the extracted WebhookHeaders.

10. **Return response** immediately.

**processEvent function** (async, inside the same file for now):
- Parse rawBody as JSON
- Add a Sentry breadcrumb with category 'webhook', message `${eventType}.${action}`, data containing deliveryId and repository full_name
- For now, this is a processing STUB -- the router and handlers are added in Plan 02
- Add a comment: `// TODO(phase-05-plan-02): Route to event handlers`
- Wrap everything in try/catch: catch calls Sentry.captureException(error)
- finally block: await flushSentry(2000) -- MUST be called per user decision to prevent Sentry data loss in serverless

**CRITICAL anti-patterns to avoid:**
- Do NOT call request.json() anywhere (consumes body, breaks signature verification)
- Do NOT return non-200 for unrecognized events (GitHub expects 200 for all valid deliveries)
- Do NOT initialize Sentry inside the fetch handler (module scope only, per user decision)
- Do NOT do any processing before returning the 200 response (response-first pattern, per user decision)
