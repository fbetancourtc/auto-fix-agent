**api/lib/router.ts** -- Event type routing:
- Import * as Sentry from '@sentry/node'
- Import ProcessEventResult from ./types
- Import { isAutoFixLabeledPR, isReviewOnAutoFixPR } from ./filters
- Import { handleWorkflowRun } from ./handlers/workflow-run
- Import { handlePullRequest } from ./handlers/pull-request
- Import { handleReview } from ./handlers/review
- `export async function routeEvent(eventType: string, payload: any): Promise<ProcessEventResult>`
- Switch on eventType:
  - 'workflow_run': if payload.action !== 'completed', add Sentry breadcrumb `Skipped workflow_run.${payload.action}`, return { processed: false, reason: 'action not completed' }. Otherwise call handleWorkflowRun(payload), return { processed: true }.
  - 'pull_request': if !isAutoFixLabeledPR(payload), add Sentry breadcrumb 'Skipped PR without auto-fix label', return { processed: false, reason: 'no auto-fix label' }. Otherwise call handlePullRequest(payload), return { processed: true }.
  - 'pull_request_review': if !isReviewOnAutoFixPR(payload), add Sentry breadcrumb 'Skipped review on non-auto-fix PR', return { processed: false, reason: 'not auto-fix PR' }. Otherwise call handleReview(payload), return { processed: true }.
  - default: add Sentry breadcrumb `Unrecognized event: ${eventType}`, return { processed: false, reason: 'unrecognized event type' }.
- All breadcrumbs use category: 'webhook.filter'
