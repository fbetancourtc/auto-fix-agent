**3. Create `scripts/record-metrics.sh`** (make executable):

The script does three things:
a) Extracts cost data from the Claude Code Action execution file (or falls back to rate table estimate)
b) Appends a run entry to `metrics/runs.json`
c) Writes a GitHub Actions summary table

**Required env vars** (set by the calling workflow step):
- `EXECUTION_FILE` — path to claude-code-action execution file output
- `TARGET_REPO` — full repo name (owner/repo)
- `RUN_ID` — the failed CI run ID
- `FIX_PR_CREATED` — "true" or "false" indicating if agent created a fix PR
- `GH_TOKEN` — for git push and gh commands
- `CENTRAL_REPO` — "fbetancourtc/auto-fix-agent" (for pushing metrics)

**Script logic:**

```bash
#!/usr/bin/env bash
set -euo pipefail

# --- Cost extraction (two-strategy approach per user decision) ---

COST=""
INPUT_TOKENS=0
OUTPUT_TOKENS=0
CACHE_READ=0
CACHE_WRITE=0
MODEL="${DEFAULT_MODEL:-claude-sonnet-4-6}"

# Strategy 1: Try parsing execution_file for total_cost_usd
if [ -n "${EXECUTION_FILE:-}" ] && [ -f "${EXECUTION_FILE:-}" ]; then
  # execution_file may be JSONL — check last line for cost fields
  COST=$(tail -1 "$EXECUTION_FILE" | jq -r '.total_cost_usd // empty' 2>/dev/null || true)
  if [ -n "$COST" ] && [ "$COST" != "null" ]; then
    INPUT_TOKENS=$(tail -1 "$EXECUTION_FILE" | jq -r '.usage.input_tokens // 0' 2>/dev/null || echo 0)
    OUTPUT_TOKENS=$(tail -1 "$EXECUTION_FILE" | jq -r '.usage.output_tokens // 0' 2>/dev/null || echo 0)
    CACHE_READ=$(tail -1 "$EXECUTION_FILE" | jq -r '.usage.cache_read_input_tokens // 0' 2>/dev/null || echo 0)
    CACHE_WRITE=$(tail -1 "$EXECUTION_FILE" | jq -r '.usage.cache_creation_input_tokens // 0' 2>/dev/null || echo 0)
  fi
fi

# Strategy 2: Fallback to rate table estimate
if [ -z "$COST" ] || [ "$COST" = "null" ] || [ "$COST" = "" ]; then
  PRICING_FILE="_auto-fix-scripts/config/pricing.json"
  INPUT_RATE=$(jq -r ".models[\"$MODEL\"].input_per_mtok" "$PRICING_FILE")
  OUTPUT_RATE=$(jq -r ".models[\"$MODEL\"].output_per_mtok" "$PRICING_FILE")
  EST_INPUT=$(jq -r '.estimated_tokens_per_run.input' "$PRICING_FILE")
  EST_OUTPUT=$(jq -r '.estimated_tokens_per_run.output' "$PRICING_FILE")
  INPUT_TOKENS=$EST_INPUT
  OUTPUT_TOKENS=$EST_OUTPUT
  COST=$(echo "$EST_INPUT $INPUT_RATE $EST_OUTPUT $OUTPUT_RATE" | awk '{printf "%.4f", ($1/1000000)*$2 + ($3/1000000)*$4}')
fi
```

    id: $id,
    timestamp: $ts,

    failed_run_id: $run_id,
    outcome: $outcome,
    cost_usd: $cost,

**1. Create `scripts/check-budget.sh`** (make executable):

**Required env vars:**
- `GH_TOKEN` — for gh issue commands
- `CENTRAL_REPO` — "fbetancourtc/auto-fix-agent" (where issues are created)

**Script logic:**

```bash
#!/usr/bin/env bash
set -euo pipefail

PRICING_FILE="_auto-fix-scripts/config/pricing.json"
METRICS_FILE="_auto-fix-scripts/metrics/runs.json"

MONTHLY_BUDGET=$(jq -r '.budget.monthly_limit_usd' "$PRICING_FILE")
CURRENT_MONTH=$(date -u +%Y-%m)
TOTAL_COST=$(jq "[.runs[] | select(.month == \"$CURRENT_MONTH\") | .cost_usd] | add // 0" "$METRICS_FILE")

PCT=$(echo "$TOTAL_COST $MONTHLY_BUDGET" | awk '{printf "%.0f", ($1/$2)*100}')

echo "Monthly spend: \$$TOTAL_COST / \$$MONTHLY_BUDGET (${PCT}%)"

# Check each threshold from config
for THRESHOLD in $(jq -r '.budget.alert_thresholds_pct[]' "$PRICING_FILE"); do
  if [ "$PCT" -ge "$THRESHOLD" ]; then
    # Dedup: check if alert already exists for this threshold + month
    EXISTING=$(gh issue list --repo "$CENTRAL_REPO" --label "budget-alert" \
      --search "Budget ${THRESHOLD}% ${CURRENT_MONTH}" \
      --json number --jq '.[0].number' 2>/dev/null || true)

    if [ -z "$EXISTING" ] || [ "$EXISTING" = "null" ]; then
      gh issue create --repo "$CENTRAL_REPO" \
        --title "Budget ${THRESHOLD}% alert - ${CURRENT_MONTH}" \
        --label "budget-alert" \
        --body "Monthly auto-fix spend has reached **${THRESHOLD}%** (\$${TOTAL_COST} of \$${MONTHLY_BUDGET}).

## Details
- Month: ${CURRENT_MONTH}
- Total cost: \$${TOTAL_COST}
- Budget: \$${MONTHLY_BUDGET}
- Threshold: ${THRESHOLD}%

## Action
$(if [ "$THRESHOLD" -ge 100 ]; then echo "Auto-fix continues running. Review spending and decide whether to pause."; else echo "No action required yet. Monitor spending."; fi)

---
*Generated by [auto-fix-agent](https://github.com/fbetancourtc/auto-fix-agent) budget monitor*"
      echo "Created budget alert for ${THRESHOLD}% threshold"
    else
      echo "Budget alert for ${THRESHOLD}% already exists (issue #${EXISTING}), skipping"
    fi
  fi
done
```

Make executable: `chmod +x scripts/check-budget.sh`

Also add a `concurrency` block to the `fix` job to serialize metrics writes (prevents concurrent corruption per Research pitfall 3):
```yaml
concurrency:
  group: auto-fix-metrics
  cancel-in-progress: false
```

**Important:** Add this concurrency at the JOB level (inside `jobs.fix:`), NOT at the workflow level. This means only one auto-fix run can write metrics at a time, but the concurrency group is shared across all repos. If a second run starts while the first is recording metrics, it waits (does not cancel).

Wait — actually, since the metrics step is at the END of the job and `cancel-in-progress: false`, this means the entire fix job for one repo would block another. That's too aggressive. Instead, do NOT add job-level concurrency. The `record-metrics.sh` script already handles concurrent pushes with a retry loop (pull-rebase-push). That's sufficient for the expected volume (~1-5 runs/day).
