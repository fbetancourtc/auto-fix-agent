  <action>
Create `.github/workflows/promote.yml` as a reusable workflow mirroring the auto-fix.yml pattern:

**Inputs:**
- `app_id` (required, string): GitHub App ID for cross-org token
- `repository` (required, string): Full repo name (owner/repo)
- `merged_pr_number` (required, string): The auto-fix PR number that was merged
- `source_branch` (required, string, default: "develop"): Branch the fix was merged into
- `qa_branch` (required, string, default: "qa"): QA branch name
- `main_branch` (required, string, default: "main"): Main branch name

**Secrets:**
- `app_private_key` (required)

**Steps (in order):**

1. **Generate token** — use `actions/create-github-app-token@v2` with `app-id` and `private-key` (same as auto-fix.yml step 1). Set `repositories` to extract repo name from `inputs.repository`.

2. **Create develop→qa PR** — Shell step using `gh` CLI:
   - Set `GH_TOKEN` env var from token output
   - Check if develop has commits ahead of qa: `gh api repos/$REPO/compare/$QA...$SOURCE --jq '.ahead_by'`
   - If ahead_by > 0, check for existing open PR: `gh pr list --repo "$REPO" --base "$QA" --head "$SOURCE" --json number --jq '.[0].number'`
   - If no existing PR, create one: `gh pr create --repo "$REPO" --base "$QA" --head "$SOURCE" --title "promote: merge $SOURCE into $QA (fix PR #$PR_NUM)" --label "auto-fix-promote" --body "..."`
   - If PR already exists, log "develop→qa PR already exists (#N), skipping"

3. **Create qa→main PR (manual merge gate)** — Shell step:
   - Check if qa has commits ahead of main: `gh api repos/$REPO/compare/$MAIN...$QA --jq '.ahead_by'`
   - If ahead_by > 0, check for existing open PR: `gh pr list --repo "$REPO" --base "$MAIN" --head "$QA" --json number --jq '.[0].number'`
   - If no existing PR, create one: `gh pr create --repo "$REPO" --base "$MAIN" --head "$QA" --title "promote: $QA to $MAIN (awaiting human approval)" --label "auto-fix-promote" --body "..."`
   - **NEVER call `gh pr merge`** — the PR is the approval gate, human must merge
   - If PR already exists, log "qa→main PR already exists (#N) — it tracks latest qa automatically"

**Key safety rules:**
- Always check for existing PRs before creating (prevents duplicates per Research pitfall 1 & 2)
- Never auto-merge qa→main (locked user decision)
- Use env var indirection for GH_TOKEN (existing pattern)
- Set `permissions: contents: read, pull-requests: write` on the job

Create `.github/workflows/promote-caller.example.yml` as thin caller template:
```yaml
# Promote Caller -- copy to your repo's .github/workflows/promote-caller.yml
# Triggers when an auto-fix PR is merged into develop.
name: Promote Fix
on:
  pull_request:
    types: [closed]
jobs:
  promote:
    if: >-
      github.event.pull_request.merged == true &&
      contains(github.event.pull_request.labels.*.name, 'auto-fix')
    uses: fbetancourtc/auto-fix-agent/.github/workflows/promote.yml@main
    with:
      app_id: "2985828"
      repository: "${{ github.repository }}"
      merged_pr_number: "${{ github.event.pull_request.number }}"
    secrets:
      app_private_key: "${{ secrets.AUTO_FIX_APP_PRIVATE_KEY }}"
```

Note: The promote caller does NOT need `anthropic_api_key` (no AI involved, just PR creation). It only needs `app_private_key` for cross-org token.
