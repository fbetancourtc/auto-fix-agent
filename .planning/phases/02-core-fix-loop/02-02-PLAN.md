**2. Create `scripts/validate-diff.sh`:**

Follow the same pattern as `scripts/sanitize-logs.sh` (existing convention).

```bash
#!/usr/bin/env bash
set -euo pipefail

# validate-diff.sh — Post-agent diff validation for FIXG-02 / FIXG-03
# Checks git diff against allowed_dirs from repo-stack-map.json.
# Reverts any forbidden file changes. Exits with status indicating result.
#
# Usage: validate-diff.sh <repo> <config-path>
# Exit codes: 0 = clean (all changes allowed or forbidden files reverted)
#             1 = no changes detected (agent produced no diff)

REPO="${1:?Usage: validate-diff.sh <repo> <config-path>}"
CONFIG="${2:?Usage: validate-diff.sh <repo> <config-path>}"

# Get allowed directories for this repo
# First check for per-repo override, then fall back to stack default
STACK=$(jq -r --arg repo "$REPO" '.repos[$repo].stack // empty' "$CONFIG")
if [ -z "$STACK" ]; then
  echo "ERROR: Repo '$REPO' not found in config"
  exit 1
fi

REPO_DIRS=$(jq -r --arg repo "$REPO" '.repos[$repo].allowed_dirs // empty | .[]?' "$CONFIG" 2>/dev/null)
if [ -n "$REPO_DIRS" ]; then
  ALLOWED_DIRS="$REPO_DIRS"
else
  ALLOWED_DIRS=$(jq -r --arg stack "$STACK" '.defaults[$stack].allowed_dirs[]' "$CONFIG")
fi

if [ -z "$ALLOWED_DIRS" ]; then
  echo "ERROR: No allowed_dirs found for repo '$REPO' (stack: $STACK)"
  exit 1
fi

# Get list of changed files (staged and unstaged)
CHANGED_FILES=$(git diff --name-only HEAD 2>/dev/null || true)
STAGED_FILES=$(git diff --cached --name-only 2>/dev/null || true)
ALL_CHANGES=$(echo -e "${CHANGED_FILES}\n${STAGED_FILES}" | sort -u | grep -v '^$' || true)

if [ -z "$ALL_CHANGES" ]; then
  echo "WARNING: No file changes detected -- agent produced no diff"
  exit 1
fi

VIOLATIONS=()
ALLOWED_COUNT=0

while IFS= read -r file; do
  MATCH=false
  while IFS= read -r dir; do
    if [[ "$file" == "$dir"* ]]; then
      MATCH=true
      break
    fi
  done <<< "$ALLOWED_DIRS"

  if [ "$MATCH" = false ]; then
    VIOLATIONS+=("$file")
    git checkout HEAD -- "$file" 2>/dev/null || true
  else
    ALLOWED_COUNT=$((ALLOWED_COUNT + 1))
  fi
done <<< "$ALL_CHANGES"

echo "Diff validation: ${ALLOWED_COUNT} allowed file(s)"

if [ ${#VIOLATIONS[@]} -gt 0 ]; then
  echo "::warning::Reverted ${#VIOLATIONS[@]} forbidden file(s): ${VIOLATIONS[*]}"
  for v in "${VIOLATIONS[@]}"; do
    echo "  REVERTED: $v"
  done
fi

if [ "$ALLOWED_COUNT" -eq 0 ]; then
  echo "WARNING: All changes were in forbidden directories -- nothing to commit"
  exit 1
fi

echo "VALID: ${ALLOWED_COUNT} file(s) in allowed directories"
exit 0
```

Make the script executable: `chmod +x scripts/validate-diff.sh`

Key design decisions (Claude's discretion from CONTEXT.md):
- Central script (follows `sanitize-logs.sh` pattern)
- Per-repo `allowed_dirs` override falls back to stack defaults
- Exit code 1 means "no valid changes" (agent fix was all in forbidden dirs or produced no diff)
- Forbidden files are reverted with `git checkout HEAD --`
- The `::warning::` syntax creates a GitHub Actions annotation

**4. Add post-agent diff validation step (new step 9):**

After the Claude Code Action step, add a diff validation step that checks the agent's pushed branch. If the agent pushed forbidden files, this step reverts them and force-pushes the correction. The PR auto-updates since it's pointing at the same branch.

```yaml
# 9. Post-agent diff validation (FIXG-02, FIXG-03)
# Checks the agent's changes against allowed directories.
# If forbidden files were committed, reverts them, amends, and force-pushes.
# If no valid changes remain after revert, closes the PR.
- name: Post-agent diff validation
  id: diff-check
  if: steps.circuit.outputs.should_proceed == 'true' && steps.flaky.outputs.is_flaky != 'true'
  env:
    TARGET_REPO: ${{ inputs.repository }}
    GH_TOKEN: ${{ steps.app-token.outputs.token }}
  run: |
    # Check if agent actually pushed a branch (it may have stopped without committing)
    BRANCH_NAME=$(git branch --show-current)
    if [[ "$BRANCH_NAME" != auto-fix/* ]]; then
      echo "Agent did not create an auto-fix branch -- no changes to validate"
      exit 0
    fi

    # Run diff validation against allowed directories
    RESULT=0
    bash _auto-fix-scripts/scripts/validate-diff.sh "$TARGET_REPO" _auto-fix-scripts/config/repo-stack-map.json || RESULT=$?

    if [ "$RESULT" -eq 0 ]; then
      echo "All changes in allowed directories -- no corrections needed"
      exit 0
    fi

    # validate-diff.sh reverted forbidden files. Check if any valid changes remain.
    REMAINING=$(git diff --name-only HEAD~1 2>/dev/null || true)
    if [ -z "$REMAINING" ]; then
      echo "::warning::All agent changes were in forbidden directories -- closing PR"
      # Find and close the PR if it exists
      PR_NUM=$(gh pr list --repo "$TARGET_REPO" --head "$BRANCH_NAME" --json number --jq '.[0].number' 2>/dev/null || true)
      if [ -n "$PR_NUM" ]; then
        gh pr close "$PR_NUM" --repo "$TARGET_REPO" --comment "Closed: all changes were in forbidden directories"
      fi
      exit 0
    fi

    # Amend commit to exclude reverted files, force-push correction
    git add -A
    git commit --amend --no-edit
    git push --force-with-lease
    echo "::warning::Reverted forbidden files and force-pushed correction"
```

This approach lets the agent do everything (fix, commit, push, PR create) naturally, then the workflow validates and corrects as needed. If the agent modified forbidden files, they are reverted and the PR auto-updates. If ALL changes were forbidden, the PR is closed.

NOTE on --force-with-lease: Safe force-push that protects against race conditions. Required because we're amending the agent's commit.
