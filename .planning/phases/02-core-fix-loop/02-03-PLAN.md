Two changes to `.github/workflows/auto-fix.yml`:

**1. Add env vars to the Claude Code Action step:**

The step (after Plan 02-02) already has:
```yaml
env:
  GH_TOKEN: ${{ steps.app-token.outputs.token }}
```

Extend it to:
```yaml
env:
  GH_TOKEN: ${{ steps.app-token.outputs.token }}
  RUN_ID: ${{ inputs.failed_run_id }}
  HEAD_BRANCH: ${{ inputs.head_branch }}
  TARGET_REPO: ${{ inputs.repository }}
```

These env vars are used by the agent (via prompt instructions) for:
- `RUN_ID`: Branch naming (`auto-fix/{RUN_ID}`) and linking to the original CI run
- `HEAD_BRANCH`: Context about which branch failed
- `TARGET_REPO`: The `--repo` flag for gh CLI commands (`gh pr list --repo "$TARGET_REPO"`)

**2. Add no-commit tracking step (after the post-agent diff validation step):**

When the agent stops without committing (fix didn't work, tests still fail), no PR is created. But per user decision, this must still count as one attempt toward the 2-attempt limit. Without tracking, the retry guard (which counts auto-fix PRs) would not see this attempt.

Add a new step that detects when the agent ran but produced no auto-fix branch/PR, and creates a lightweight tracking marker:

```yaml
# 11. Track no-commit attempt (Warning 1 fix)
# If the agent ran but did not create an auto-fix branch (no commit/push/PR),
# this still counts as one attempt. Create a closed "tracking" PR so the
# retry guard's gh pr list count includes it.
- name: Track failed attempt
  id: track-attempt
  if: steps.circuit.outputs.should_proceed == 'true' && steps.flaky.outputs.is_flaky != 'true'
  env:
    GH_TOKEN: ${{ steps.app-token.outputs.token }}
    RUN_ID: ${{ inputs.failed_run_id }}
    TARGET_REPO: ${{ inputs.repository }}
    HEAD_BRANCH: ${{ inputs.head_branch }}
  run: |
    # Check if agent created an auto-fix branch
    BRANCH_NAME="auto-fix/${RUN_ID}"

    # If a PR already exists for this branch, the agent succeeded -- nothing to track
    EXISTING_PR=$(gh pr list --repo "$TARGET_REPO" --head "$BRANCH_NAME" --state all --json number --jq '.[0].number' 2>/dev/null || true)
    if [ -n "$EXISTING_PR" ]; then
      echo "Agent created PR #${EXISTING_PR} -- no tracking needed"
      exit 0
    fi

    # Check if the branch exists on the remote (agent pushed but maybe didn't create PR)
    git ls-remote --exit-code origin "$BRANCH_NAME" 2>/dev/null && {
      echo "Branch exists on remote but no PR -- agent may have been interrupted"
      exit 0
    }

    # Agent did not push anything -- create a tracking branch with an empty commit
    # This creates a closed PR that the retry guard will count
    git checkout -b "$BRANCH_NAME"
    git commit --allow-empty -m "fix: auto-fix attempt failed (no viable fix found)

Run: ${RUN_ID}
Branch: ${HEAD_BRANCH}
Status: Agent could not produce a working fix"
    git push origin "$BRANCH_NAME"

    # Create and immediately close a PR so retry guard counts this attempt
    PR_URL=$(gh pr create --repo "$TARGET_REPO" \
      --base develop \
      --head "$BRANCH_NAME" \
      --label "auto-fix" \
      --title "fix: auto-fix attempt failed (run #${RUN_ID})" \
      --body "## Failed Auto-Fix Attempt

The agent analyzed the CI failure but could not produce a working fix.
This PR is created for tracking purposes and is immediately closed.

**CI Run:** [#${RUN_ID}](https://github.com/${TARGET_REPO}/actions/runs/${RUN_ID})
**Branch:** \`${HEAD_BRANCH}\`

---
*Generated by [auto-fix-agent](https://github.com/fbetancourtc/auto-fix-agent)*")

    gh pr close "$PR_URL" --repo "$TARGET_REPO" --comment "Auto-closed: tracking a failed fix attempt (agent could not produce a working fix)"
    echo "Tracked failed attempt via closed PR: $PR_URL"
```

This ensures the retry guard (which counts `auto-fix` labeled PRs via `gh pr list --label auto-fix --state all`) correctly includes failed attempts where the agent stopped without committing. The closed PR is a lightweight marker visible in the PR history.

The new sections are IDENTICAL across all three stacks (they are stack-agnostic). Add the following content to each prompt file:

```markdown
## Before You Start

Before implementing any fix, you MUST complete these checks in order:

### 1. Check Retry Count

Run this command to count previous auto-fix attempts for this failure:

```bash
ATTEMPT_COUNT=$(gh pr list --repo "$TARGET_REPO" --label auto-fix --state all \
  --json headRefName \
  --jq '[.[] | select(.headRefName | startswith("auto-fix/"))] | length')
echo "Previous auto-fix attempts: $ATTEMPT_COUNT"
```

- If `ATTEMPT_COUNT >= 2`: Do NOT attempt a fix. Skip to the **Escalation** section below.
- If `ATTEMPT_COUNT < 2`: Proceed with the fix.

### 2. Create Fix Branch

```bash
git checkout -b "auto-fix/${RUN_ID}"
```

This creates the fix branch from the current HEAD (which is the failing branch, checked out by the workflow).

## Fix Workflow

After diagnosing the failure and implementing your fix:

1. Run the appropriate test/lint commands to verify your fix works
2. If tests still fail after your fix, **stop** -- do not commit or push. The workflow will detect the no-commit outcome and count it as a failed attempt.
3. If tests pass, stage, commit, push, and create PR:

```bash
git add -A
git commit -m "fix(<scope>): <brief description of the fix>

Root Cause: <one-line explanation of why CI failed>
Changes: <one-line summary of what was changed>
Verified: <which test/lint command confirmed the fix>"

git push origin "auto-fix/${RUN_ID}"

gh pr create --repo "$TARGET_REPO" \
  --base develop \
  --head "auto-fix/${RUN_ID}" \
  --label "auto-fix" \
  --title "fix(<scope>): <brief description>" \
  --body "## Root Cause Analysis

<Explain why CI failed -- what was the root cause of the test/lint/build failure>

## Changes Made

<List the files changed and what was modified in each>

## Verification Results

<Which test/lint commands were run and their results>

---
**CI Run:** [#${RUN_ID}](https://github.com/${TARGET_REPO}/actions/runs/${RUN_ID})
**Branch:** \`${HEAD_BRANCH}\`
*Generated by [auto-fix-agent](https://github.com/fbetancourtc/auto-fix-agent)*"
```

Use conventional commits format. The `<scope>` should be the module or feature area (e.g., `auth`, `api`, `ui`, `lint`).

**PR Description Requirements:** The PR body MUST include three sections:
1. **Root Cause Analysis** -- Why CI failed
2. **Changes Made** -- What files were changed and why
3. **Verification Results** -- Which commands confirmed the fix works

## Escalation

If the retry count check showed `ATTEMPT_COUNT >= 2`, create an escalation issue instead of attempting a fix:

```bash
# Get links to previous auto-fix PRs
PREV_PRS=$(gh pr list --repo "$TARGET_REPO" --label auto-fix --state all \
  --json number,url,headRefName \
  --jq '[.[] | select(.headRefName | startswith("auto-fix/")) | "- PR #\(.number): \(.url)"] | join("\n")')

# Read sanitized CI logs
CI_LOGS=$(cat /tmp/ci-logs.txt 2>/dev/null || echo "CI logs not available")

gh issue create --repo "$TARGET_REPO" \
  --label "needs-human" \
  --title "CI fix failed after 2 attempts: ${HEAD_BRANCH}" \
  --body "## Auto-Fix Escalation

**Status:** 2 fix attempts failed -- needs human review

### Failure Context
- **Branch:** \`${HEAD_BRANCH}\`
- **CI Run:** [#${RUN_ID}](https://github.com/${TARGET_REPO}/actions/runs/${RUN_ID})

### Previous Fix Attempts
${PREV_PRS}

### CI Failure Logs
<details>
<summary>Full sanitized CI logs (click to expand)</summary>

\`\`\`
${CI_LOGS}
\`\`\`

</details>

---
*Escalated by [auto-fix-agent](https://github.com/fbetancourtc/auto-fix-agent)*"
```

After creating the escalation issue, **stop**. Do not attempt any fix.
```
